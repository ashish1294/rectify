About Rectify

Rectify Round 1

Rectify is an individual event which tests not only the mastery and familiarity with a programming language but also the contestant's dexterity in analyzing problems and providing logical solutions.
The first round is a pencil and paper round where the participants are given a set of small snippets of source code and are expected to give the output. All questions are based on basic knowledge of C or C++.

Rectify Round 2

The second round involves debugging the given code. Here you will be given a code along with expected output and a short description of the code's functionality. All codes are written in C-style C++. You are expected to find the bug in the given code, correct it and submit it within the given time frame. Your submitted code is judged against multiple internal test cases to determine if your solution was correct.

Rectify Final Round

The final round is modeled like a top coder challenge round. The contest has 2 major phases. In the first of the two phases (Coding Phase), contestants are given problem statements for which they have to come up with a solution. Once the time for submitting solution finishes i.e, after the end of coding phase, the second phase (Challenge / Hack Phase) starts where the contestant will have access to the solutions submitted by other participants and they can challenge the solutions with an their own input - for which they think the solution will give wrong answer. If the other contestant's solution gives wrong answer then the challenger gains points and the challenged contestant loses points. If the challenge fails then the challenger loses points. Finally everyone's solution which are still standing are tested by the system and they gain points if they pass system test. The one with the most points wins.
